Система для предупреждения пользователей о неблагоприятных погодных изменениях.

Процесс создания:

1. Сбор данных. Поскольку нужно учитывать определённые условия (температура не меньше -3C и скорость ветра не больше 15MPH), для каждого аэропорта собранные данные сначала чистились (отбирались нужные числа), а потом проходили проверку и, в соответствии с погодными условиями, записывался и код (всё плохо/всё нормально).
2. Проектирование (планирование) вида сервиса. Было решено, что в приложении, которое состояло бы из нескольких страниц (для каждого аэропорта), было бы не очень удобным: было бы желательно сразу же увидеть аэропорты, в которых неблагоприятные погодные условия, чем в незнании переходить на 29 страниц с малым количеством информации в каждой.
Так, сервис состоит всего из одной страницы, где пользователю представляется список с аэропортами (в начале списка выделены аэропорты с плохими погодными условиями) и краткая сводка о том, сколько аэропортов сейчас переживает подобное.
3. Дополнительные манипуляции. Поскольку для такого сервиса некоторые полученные данные будут необязательны (имеется в виду код состояния, тк оно отмечено цветом), некоторые данные были убраны; также, был совершён подсчёт аэропортов с неблагоприятными погодными условиями.
4. Сам же веб-сервис должен находиться на http://donkey1.nosoc.io:5008 (но у меня не получилось настроить nginx; мануально же страничка успешно запускается)

Приложенные файлы:
airports.txt: список предложенных аэропортов. Один из аэропортов в таблице повторяется дважды (KW99), поэтому их не 30, а 29;
script.sh: парсинг данных и файл, поставленный на crontab. Собирает все данные (температура, ветер, состояние) в  info.csv;
info.csv: генерируемый script.sh датасет;
make_html.py: скрипт, создающий html-template. Присутствует здесь для того, чтобы сделать внешний вид таблицы приемлемым;
templates/page.html: генерируемая make_html.py страница;
app.py - файл с запуском приложения.

Важно! В случае использования приложения рекомендуется назначить папку проекта своей рабочей директорией (чтобы можно было без проблем прочитать список аэропортов, сохранять/извлекать данные и создать шаблон страницы (например, начиная путь с "~/").
(В скриптах, заархивированных непосредственно с самого unix, пути прописаны так, как если бы код находится на unix в папке "homework" моей (xenak) директории).
